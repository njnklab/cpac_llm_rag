# RAGAS（Ollama-only）检索策略小实验：向量 only vs BM25 only vs 0.7/0.3 融合
> 用于硕士论文中的“知识库检索质量与证据一致性（RAG 组件级）评估”小实验  
> 场景：LLM+RAG 生成 CPAC 预处理参数配置建议（含 YAML 片段 + 解释）

---

## 1. 实验目标与结论输出
### 1.1 目标
从指定 query 文件中抽取少量问题，在固定知识库、固定生成模型与 prompt 的条件下，对比三种检索策略：
1) vector_only（向量 only）
2) bm25_only（BM25 only）
3) fusion_0.7_0.3（向量 0.7 + BM25 0.3 融合）

对每个策略，生成 CPAC 参数建议输出，并用 RAGAS 计算 **5 个 reference-free 指标**，最后绘制：
- 5 个指标的 **均值（mean ± std）** 对比图
- 5 个指标的 **分布图（boxplot 或 violin）** 对比图

### 1.2 输出文件（强制）
每个 mode（检索策略）输出：
- `samples_{mode}.jsonl`：每条 query 一行（含 user_input / retrieved_contexts / response）
- `ragas_{mode}_per_sample.csv`：每条样本的 5 个指标分数
- `ragas_{mode}_overall.json`：5 个指标整体 mean/std/median/p10/p90

总图输出：
- `fig_mean_5metrics.png`：每个指标三根柱（或点线）+ errorbar
- `fig_dist_5metrics.png`：5 个指标 × 3 策略 的分布图（建议 5 张子图或拼接）

### 2.1 抽取规则（建议最简单可复现）
- 从每个文件读取所有非空行（strip 后长度>0）
- 取前 N 条作为测试（推荐 N=5；总计 10 条 query）
- 若想随机抽取：固定 seed（例如 2026），从每个文件随机采样 N 条

> **本实验建议先用 N=5**，能跑通并出图即可；后续要更稳健再扩大 N。
